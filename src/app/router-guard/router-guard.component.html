Angular Route guards can be used to controll whether the user can navigate to or away from the given route based on some condition.

Use case of Route Guards

To confirm the navigation operation

Asking whether to save data before moving away from view

Allow access to certain parts of application to specific users

Validating the Route parameters before navigating to the route

Fetching some data before displaying the component view

Types of Route Guards

  • CanActivate:This help to decide whether the route can be activated or not.
  • CanDeactivate:This help to decide whether the route can be activated or not; generally, this is used to warn a user if they are navigating from the component.
  • Resolve:This performs route data retrieval before any activation of the route.
  • CanLoad:Checks to see if the user can navigate to the module which is lazily loading.
  • CanActivateChild:This help to decide whether the child route can be activated or not.

    • We can use angular CLI command to create a guard: ng g g [guard name]

    How to create a Route Guard

  • Creating a route guard as a service.
  • Implementing the respective route guard interface and the required router guard method.
      • Within this method a boolean value is returned.
      If we return true for this method in that case navigation will continue.
      If we return false for this method in that case the navigation process will stop.
  • Use the route guard on the route which we want to guard.

    Can Deactivate Route Guard

    The best use case where we can use CanDeactivate route guard is, when the user has filled a form like registration form and tries to navigate away without submitting. In that case, before navigating the user away, we might want to confirm if he want to submit the data.

    On clicking on the back button

    Without filling out the full form and submitting we cannot rout to any page. There will be confirm window open. After submiting we can go back.

    This functionality is achieved with the help of CanDeactivateGuard

    Lazy Loading

    Lazy loading is a technique in Angular that allows modules to be loaded on-demand, improving application performance by only loading the modules when they are needed.

    Lazy loading is a module on demand. Lazy loading module help us decrease the startup time. With lazy loading our application doesnot need to load everything at once, it only need to load what the user expected to see when the app first loads. Modules thar are lazily loaded will only be loaded when the user navigates to their routes.

    Inside lazy loading module we can have:

  • -standalone component
  • -regular component --standalone false

    • To generate a lazy loading module ng g module moduleName --routing

    Creating component add-user ng g c router-guard/admin/add-user

    Creating component edit-user ng g module router-guard/admin/edit-user

    As Such we are not seeing any difference in view but the difference is in performane. Without lazy loading the entire application will first load then comes in view.

    But with lazy loading on clicking on Add user the add-user component loads.
