Why not use normal HTML Form?

When a normal html form is submitted, it reloads the page by making an HTTP request to the server. Since we create a single page application using angular, we can say that it will start the complete angular application.

To avoid this, we need angular help to stop this default behaviour. This can be achived using template driven form or reactive form in Angular. Using any one of these two approach makes working with forms easier.

There are two types of form

  • Template-Driven Form
      • Simple basic form
      Easy to start with
      Vased on template (HTML)
  • Reactive Form
      • Complex form with more control
      Structure of form is defined in Typescript class

    • Touched and Dirty Form Property

    A form is considered as touched if any of its control has been focused. For a touched form its touched property will be true and its untouched property will be false.

    A form control is considered as touched if that form control has been focused. For a touched form control its untouched property will be false.

    We want to add form-shadow class dynamically when the form is touched

    For this we use ngClass directive

    Now on the container: [ngClass]="{{back}}'form-shadow': myForm.touched{{front}}"

    A form is considered dirty when something typed in the form.

    We want to add class form-background based on dirty.

    Now on the container: [ngClass]="{{back}}'form-shadow': myForm.touched{{front}}, 'form-background': myForm.dirty{{front}}"

    Now when we click on any of the field and click outside the form-shadow class is triggered.
